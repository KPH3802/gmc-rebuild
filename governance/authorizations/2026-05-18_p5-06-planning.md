## Candidate P5-06 Shape (Planning Level Only — Future / Not Authorized)

This section records, at the planning level only, the conservative candidate shape for a future P5-06 packet. Recording this shape **does not authorize implementation**. Kevin must independently authorize any P5-06 implementation packet in writing, in a separate authorization artifact under `governance/authorizations/`, with its own scope, constraints, and validation requirements; that future authorization is free to replace any element of the candidate shape recorded here.

**Recommended candidate shape (tripwire-only, no new production behaviour).** A bounded **tests-only tripwire packet**, modeled directly on the merged P5-03 (`tests/simulation/test_simulated_order_intent_invariants.py`), P5-04 (`tests/simulation/test_composed_safety_foundation.py`), and P5-05 (`tests/runtime/test_operator_view_composed_safety_foundation.py`) precedents. It would add exactly one new test file under the existing `tests/` tree, exercising only the **already-merged** P2 / P3 / P4 / P5 public surfaces. It would add no new `src/**` directory, modify no `src/**` file, modify no existing test file, add no new public symbol, add no `SimulationLane` / `SimulatedOrderSide` / `SimulatedOrderType` member, add no field on `SimulatedIntent` or `SimulatedOrderIntent`, add no method on `SimulationBoundary` beyond the merged `propose` (P5-01) and `propose_order` (P5-02), add no placeholder or order record class, and preserve the §8 step 4a `allowed_p2_infra` allowlist exactly at ten entries.

Within that tripwire-only envelope, several plausible directions exist for a future P5-06 authorization to choose from. **None of these is pre-authorized**; they are listed at the planning level only so that a future P5-06 authorization can name the chosen direction explicitly:

- **Direction (a) — symmetric-property tripwire across `propose` and `propose_order`.** A focused test module that asserts symmetric structural properties between `SimulationBoundary.propose` (P5-01) and `SimulationBoundary.propose_order` (P5-02) — e.g., identical lane-gating behavior, identical `SafetyVerdict.clear` precondition surfacing, identical blocker-code propagation shape, identical non-mutation contract — under the real composed P4-06 / P4-07 / P4-08 pipeline. Complements the merged P5-04 (composed safety × simulation) and P5-05 (composed operator-view × safety × simulation) tripwires by adding a `propose` ↔ `propose_order` symmetry tripwire.
- **Direction (b) — operator-view determinism / idempotence under the real composed pipeline.** A focused test module that asserts that `format_safety_verdict` is byte-identical across repeated calls for the same `SafetyVerdict` (deterministic rendering tripwire), and that repeated calls under the real composed P3 / P4-06 / P5-01 / P5-02 pipeline do not perturb any observable state in the P3 fakes, the safety surface, or the simulation surface. Tightens the P5-05 determinism property under repeated rendering.
- **Direction (c) — closed-enum / closed-field defense-in-depth tripwire.** A focused test module that re-asserts, under the real composed pipeline, the closed-set invariants of `SimulationLane` (one member: `LOCAL_ONLY`), `SimulatedOrderSide` (two members: `BUY`, `SELL`), `SimulatedOrderType` (two members: `MARKET`, `LIMIT`), and the exact eight fields of `SimulatedOrderIntent`. Adds defense-in-depth against future enum-expansion / field-addition drift by checking the closed surface from a composed-pipeline vantage point.
- **Direction (d) — ADR-005 / ADR-002 / ADR-003 composed-policy alignment tripwire.** A focused test module that re-asserts ADR-005 (operator-availability heartbeat safe-default — STALE for unknown components), ADR-002 (kill-switch ARMED biconditional), and ADR-003 (reconciliation CLEAN biconditional) at the composed-pipeline level, ensuring the merged P4-06 `RuntimeShell` continues to honor each ADR's safe-default rule end-to-end under the merged P3-03 / P3-04 / P3-05 fakes.

**Out of candidate scope** (forbidden at the always-forbidden category level per `MASTER_STATUS.md` §6 independent of phase, and re-forbidden by this planning packet's constraints):
