# P5-02 Authorization — Simulated Order Intent Model

Date: 2026-05-17
Authorizer: Kevin Heaney
Scope: Bounded, narrowly-shaped P5-02 implementation packet for `gmc_rebuild` adding the **second Phase 5 implementation task** — a simulated order intent model added on top of the merged P5-01 inert local simulation boundary skeleton (`governance/authorizations/2026-05-17_p5-01.md`). P5-02 declares two new closed string enums (`SimulatedOrderSide` with `BUY` / `SELL` only, `SimulatedOrderType` with `MARKET` / `LIMIT` only), an immutable `SimulatedOrderIntent` frozen-dataclass record carrying lane / intent_id / created_at / symbol / side / quantity / order_type / limit_price, and a new `SimulationBoundary.propose_order` method that gates the new record on the existing P4-06 / P4-07 / P4-08 safety foundation (`SafetyVerdict.clear`) using the same identity-return contract as the existing `SimulationBoundary.propose`. No new module file is added; the new surface lives entirely within the existing `src/gmc_rebuild/simulation/` subpackage (extending `_boundary.py` and re-exporting from `__init__.py`). No new directory under `src/**` or `tests/**` is created. The `MASTER_STATUS.md` §8 step 4a `allowed_p2_infra` bash gate (synced to ten entries by GOV-01) is preserved unchanged.

## Allowed Scope

The implementation PR may include exactly:

- An extension of `src/gmc_rebuild/simulation/_boundary.py` adding:
  - `SimulatedOrderSide` — a closed `StrEnum` with exactly two members: `BUY = "buy"` and `SELL = "sell"`. No other side is declared. The enum is intentionally not future-extensible without separate written authorization; the matching test `tests/simulation/test_simulated_order_intent.py::test_simulated_order_side_has_only_authorized_values` asserts `members == {"buy", "sell"}` as a tripwire.
  - `SimulatedOrderType` — a closed `StrEnum` with exactly two members: `MARKET = "market"` and `LIMIT = "limit"`. No other order type (`STOP`, `STOP_LIMIT`, `MARKET_ON_CLOSE`, `LIMIT_ON_OPEN`, `TRAILING_STOP`, etc.) is declared. The matching tripwire test asserts `members == {"market", "limit"}`.
  - `SimulatedOrderIntent` — a frozen, slotted `@dataclass(frozen=True, slots=True)` with exactly eight fields in this order: `lane: SimulationLane`, `intent_id: str`, `created_at: str`, `symbol: str`, `side: SimulatedOrderSide`, `quantity: int`, `order_type: SimulatedOrderType`, `limit_price: float | None`. The dataclass declares **no** other field; `tests/simulation/test_simulated_order_intent.py::test_simulated_order_intent_dataclass_fields_are_exactly_the_authorized_set` asserts the field tuple exactly as a structural tripwire.
  - A `__post_init__` on `SimulatedOrderIntent` performing strict per-field validation: `lane` must be a `SimulationLane`; `intent_id` must be a non-empty whitespace-free `str`; `created_at` must be a non-empty `Z`-suffixed UTC string; `symbol` must be a non-empty whitespace-free `str`; `side` must be a `SimulatedOrderSide`; `quantity` must be an `int` (not `bool`, not `float`) and strictly positive; `order_type` must be a `SimulatedOrderType`; `limit_price` must be `None` when `order_type` is `MARKET` and must be a positive finite `float` when `order_type` is `LIMIT` (`bool`, `int`, `nan`, `inf`, `-inf`, zero, and negative values are explicitly rejected for `LIMIT`).
  - A `SimulatedOrderIntent.build(*, lane, intent_id, created_at, symbol, side, quantity, order_type, limit_price=None)` classmethod that accepts a UTC `datetime` for `created_at` and converts it via `gmc_rebuild.risk.to_utc_string` (the same pattern used by `SimulatedIntent.build`).
  - A new `SimulationBoundary.propose_order(*, order_intent, verdict) -> SimulatedOrderIntent` method that returns the supplied intent **by identity** if and only if (a) `order_intent.lane is self.lane` and (b) `verdict.clear is True`. Any other condition raises `SimulationBoundaryError` with the verdict's `blockers` tuple surfaced in the error message. The existing `SimulationBoundary.propose` method for `SimulatedIntent` is preserved unchanged; `propose_order` is strict about its input type and rejects a P5-01 `SimulatedIntent` passed in by mistake.
  - An updated `__all__` listing exactly seven public names: `SimulatedIntent`, `SimulatedOrderIntent`, `SimulatedOrderSide`, `SimulatedOrderType`, `SimulationBoundary`, `SimulationBoundaryError`, `SimulationLane`.
  - The module's permitted imports remain exactly `__future__`, `dataclasses`, `datetime`, `enum`, `gmc_rebuild.risk` (for `to_utc_string` only), and `gmc_rebuild.runtime` (for `SafetyVerdict` only). The module continues to contain no `__main__`, no `if __name__`, no `time.sleep`, no `asyncio.sleep`, no `os.environ`, no `os.getenv`, no `open(`, no `socket.`, no `urllib.`, no `requests.`, no broker SDK, no market-data feed, no persistence, no scheduler, and no daemon. - An update to `src/gmc_rebuild/simulation/__init__.py` extending the module docstring to reference the P5-02 authorization and adding `SimulatedOrderIntent`, `SimulatedOrderSide`, and `SimulatedOrderType` to both the import block and the `__all__` list.
