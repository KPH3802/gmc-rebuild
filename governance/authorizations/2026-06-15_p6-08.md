## Authorized Contract

### 1. Test Layout (Authorized)

- **Location:** new sibling test directory `tests/p6_08_failure_handling/` with two files: an empty `__init__.py` and a single tripwire module `test_phase_6_failure_modes.py`. The directory name `p6_08_failure_handling` tokenizes to `p6`, `08`, `failure`, `handling` — none of which appears in the §8 forbidden-token set; the §8 step 4 / step 4c scan stays clean.
- **No per-layer file split.** The planning packet §3.2 surfaced two layouts; this implementation authorization picks the single-file layout to avoid any `signal_intake_failure_modes.py` filename that would trip the §8 step 4c bare `signal` token. (The bare `signal` token would be an authorized, expected flag per the merged P6-01 `signal_intake` precedent, but the single-file layout is the lower-noise choice.)
- **Helper-only module.** All test helpers (`_expect`, `_expect_match`, `_clear_verdict`, `_blocked_verdict_kill_switch`, `_config`, `_signal`, `_would_trade_decision`, `_would_skip_decision`, `_order_intent`, `_build_clear_runtime_shell`, `_walk_p6_source_files`, `_collect_imports_from_this_module`) are private to the new module; no helper is exported, no helper module is added to a package `__init__.py`, and no existing test file is modified.
- **No `pytest.raises`.** Exception expectations use the in-repo `_expect` / `_expect_match` try/except helpers, matching the merged P6-NN test convention (none of which imports `pytest` at module level).
