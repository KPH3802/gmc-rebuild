## Allowed Scope

The P3-04 implementation PR, under this authorization, may include exactly:

- A new pure-Python `src/gmc_rebuild/kill_switch/` package containing:
  - `__init__.py` — package marker, exporting only the in-memory fake fixture type. No runtime side effects on import.
  - `_fake.py` — a sandboxed in-memory fake implementation of `KillSwitchProtocol` whose sole purpose is to be exercised by tests under `tests/kill_switch/`. Pure Python, deterministic, no I/O, no network, no `time.sleep`, no `os.environ` / `os.getenv`, no filesystem, no broker SDK, no scheduler, no background thread, no daemon, no `__main__` entry point, no secrets, no real account / venue / endpoint identifier, no order placement, no broker activity, no market data.
- A new `tests/kill_switch/` package containing:
  - `__init__.py` — test package marker.
  - `test_kill_switch_fixture.py` — focused, deterministic tests for the in-memory fixture, asserting `KillSwitchProtocol` shape conformance via `isinstance` against the `runtime_checkable` Protocol, plus a small number of behavioural invariants for the fixture (fresh construction is ARMED with empty reason / triggered_by; a trip records TRIPPED with the supplied reason / triggered_by; subsequent `current()` calls remain TRIPPED; a second trip overwrites the recorded values; `current()` does not raise in the steady-state case; ADR-004 UTC discipline is preserved at the boundary; the fixture has no broker / network / filesystem coupling).
- A `MASTER_STATUS.md` §8 step 4a allowlist extension that adds **exactly** `src/gmc_rebuild/kill_switch` to `allowed_p2_infra`, with a matching comment block citing this authorization artifact and the P3-04 implementation PR. **No other path is added to the allowlist.** The new `tests/kill_switch/` directory does **not** require an allowlist entry because the §8 step 4a allowlist is scoped to **Phase 2 / Phase 3 infrastructure paths under `src/**`** only (the §8 step 4a scan in `MASTER_STATUS.md` only iterates over candidate `src/**` infrastructure paths; `tests/**` is governed by `pyproject.toml` `testpaths` and the always-forbidden token scan in §8 step 4c, not by the step 4a allowlist).
- Governance prose updates limited to: `MASTER_STATUS.md` §1, §5 (P3-04 implementation entry), §6 (current next allowed decisions), §7 (Kevin's decision record), §8 (P3-04 implementation references and the `src/gmc_rebuild/kill_switch` allowlist entry), §9 (next allowed decisions), `README.md` "Current Phase" cross-reference, `plan/phase3_entry_plan.md` §1 "Current Status", and the package-skeleton test `tests/test_package_skeleton.py` (which enumerates the authorized direct children of `src/gmc_rebuild/`) — narrowly recording that P3-04 implementation has merged for the KillSwitchProtocol fixture only and that no other Phase 3 implementation is authorized.
- This authorization artifact at `governance/authorizations/2026-05-14_p3-04.md`.

The implementation must, by content:

1. Preserve the P2-05 boundary unchanged. `src/gmc_rebuild/risk/` continues to contain only the abstract `typing.Protocol` definitions and supporting frozen dataclasses / enums. The in-memory fixture lives under a separate `src/gmc_rebuild/kill_switch/` package; it does **not** modify any file under `src/gmc_rebuild/risk/`.
2. Be a structural fixture only — the in-memory fake records at most one active trip against a fixed in-memory clock and conforms structurally to `KillSwitchProtocol` so `isinstance(..., KillSwitchProtocol)` is `True`. Calls to `current()` do not raise in the steady-state case, preserving ADR-002's runtime contract. A freshly constructed fake is `ARMED`; once `trip` is recorded the fake stays `TRIPPED` until discarded, with no clear / reset API (ADR-002's reset workflow is out of scope for this test-fixture artifact).
3. Avoid any runtime activation. The package is not imported by any `__main__` entry point, by any daemon, by any scheduler, by any broker integration, by any real-runtime path, and is not re-exported by `src/gmc_rebuild/__init__.py`. Its only consumers are the test fixtures themselves.
4. Preserve the P3-03 HeartbeatProtocol fixture unchanged. No file under `src/gmc_rebuild/heartbeat/` or `tests/heartbeat/` is modified.
5. Be reversible by a single `git revert` on the merge commit.
