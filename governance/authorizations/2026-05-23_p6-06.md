## Authorized Implementation Shape (For the Future P6-06 Implementation PR)

The future P6-06 implementation PR — opened separately, from clean `main`, on its own feature branch — is authorized to make only the following changes, within the contract below. Each numbered item resolves a choice the planning packet deferred.

### 1. Location — New Sibling Subpackage (Authorized)

`src/gmc_rebuild/reporting/` (with tests under `tests/reporting/`), mirroring the merged P6-01..P6-05 package precedent: an `__init__.py` re-export plus a single internal module (e.g. `_report.py`). This resolves planning §3.2 in favor of **location A**. The in-place extension of `src/gmc_rebuild/logging/` (location B) is **not** authorized — keeping the report in its own subpackage leaves the merged P2-04 surface byte-for-byte unchanged and isolates the capability for a single-`git revert` rollback. The candidate name `reporting` does **not** collide with the §8 forbidden-token set (`strategy strategies signal signals scanner scanners model models portfolio backtest backtests broker brokers execution executions live paper daemon daemons market_data order orders secret secrets`), so the §8 step 4 / 4c scan stays clean with no reconciliation note (a positive contrast to `signal_intake` and `portfolio_state`). The implementation PR must name the directory by exact path and must not choose a colliding synonym.

### 2. Audit Category — Reuse the Closed `lifecycle` Category (Authorized)

The report is emitted via `audit_event("lifecycle", "lifecycle.daily_report", <message>, fields={...}, timestamp=<caller-supplied>)`, reusing the merged **closed** `AUDIT_CATEGORIES = {governance, config, lifecycle}` and the `<category>.<action>` name rule. This resolves planning §3.4 in favor of **option 1**. **No** change to `AUDIT_CATEGORIES` or any other part of the merged P2-04 `audit_event` / `serialize_event` / `AuditEvent` surface is authorized; P6-06 is a pure *consumer* of the frozen P2-04 contract. A new audit category (option 2) is **not** authorized.

### 3. Record Shape — Frozen Value Object Rendered to an `AuditEvent` (Authorized)

A frozen, slotted `DailyReport` value object (planning §3.3 shape (1)) holding a small closed set of summary fields, plus a pure builder that constructs it from the cycle inputs (§5) and a pure renderer that maps it to an `AuditEvent` (and, if useful, its deterministic `serialize_event` JSON). The frozen/slotted discipline mirrors the merged P6-01..P6-05 dataclasses. The exact field set and signatures are named by the implementation PR within this contract; they must remain a closed, value-typed summary (no open-ended mutable containers).

### 4. §8 Allowlist Impact (Authorized for the Implementation PR)

The implementation PR adds **exactly one** new §8 step 4a `allowed_p2_infra` entry (`src/gmc_rebuild/reporting`), plus its `pr_tag` arm, iteration entry, and comment block, **in the same PR that introduces the directory**, per `MASTER_STATUS.md` §8 step 4b. **This authorization packet does not add the entry** (no directory exists yet).

### 5. Closed Inputs (Authorized)

All inputs are local, deterministic, value-typed, and caller-supplied — no env var, no filesystem, no broker session, no market-data snapshot, no real account, no clock read:

- The cycle's P6-03 `PositionDecision` values (a tuple), consumed unchanged, to summarize counts (e.g. would-trade vs would-skip) and applied intent IDs.
- The end-of-cycle P6-05 `SimulatedPortfolio` snapshot, consumed unchanged, to summarize the position book.
- A reconciliation status value (the merged P3-05 / P2-05 `ReconciliationStatus`), preserving the `UNAVAILABLE` vs `FAILED` distinction end-to-end per ADR-003.
- A caller-supplied closed tuple of tripped-invariant codes (strings), never inferred from any external source.
- An **explicit caller-supplied timezone-aware UTC `datetime`** used as the `audit_event` timestamp, plus a caller-supplied report/cycle date label.

The implementation PR names the exact input types within this set.

### 6. Closed Outputs (Authorized)

A frozen, slotted `DailyReport` value object summarizing the cycle (date label; decision counts; applied-intent-ID tuple; position summary; reconciliation status; tripped-invariant tuple) **and** a deterministic `AuditEvent` rendered via the merged `audit_event` helper. **No** P&L, valuation, cash ledger, balances, broker reconciliation, live account, external state, or on-disk artifact. Hashability is inherited from carried values (the P6-03 / P6-05 precedent) or chosen as an explicitly hashable closed representation if named.

### 7. Required Implementation Constraints (Authorized Contract)

The implementation must be **local-only, deterministic, pure / value-typed, in-memory, side-effect free**. It must have: **no** scheduler, **no** daemon, **no** broker, **no** account identifier, **no** network, **no** persistence / database / on-disk artifact / filesystem write (`open(`, `pickle`, `shelve`), **no** market data, **no** secrets, **no** env-var read, **no** `__main__` / runtime activation, **no** external log sink / `logging.basicConfig` / handler installation, **no** tag/release, and **no** clock read (the timestamp is caller-supplied per §5). No mutation of inputs; no hidden/module-level mutable state. Closed import surface: only `__future__`, stdlib `dataclasses` / `enum` / `typing` / `collections.abc` as needed, the merged `gmc_rebuild.logging`, and the already-merged in-repo modules required to type the inputs (`gmc_rebuild.decision`, `gmc_rebuild.portfolio_state`, and the reconciliation-status type's home module); no `os`, `socket`, `requests`, `urllib`, `http`, `threading`, `asyncio`, `subprocess`, `sqlite3`, `time`, `pickle`, `shelve`, `ssl`, `smtplib`, `ftplib`, `uuid`, `random`, or any third-party dependency. Not re-exported from `src/gmc_rebuild/__init__.py`.

## Required Tests (For the Future P6-06 Implementation PR)

At minimum, focused deterministic tests covering:

- **Determinism** — identical inputs + the same explicit timestamp yield the byte-for-byte identical `DailyReport`, `AuditEvent`, and `serialize_event` JSON across repeated calls.
- **Idempotence under repeated rendering** — re-rendering the same report yields equal `AuditEvent`s; no accumulation.
- **Explicit-timestamp / no-clock-read** — the builder/renderer requires a caller-supplied timestamp, two calls with the same timestamp match, and an inertness self-check confirms no `now_utc()` / `time.*` / `datetime.now()` call.
- **Audit-contract conformance** — category in the closed `AUDIT_CATEGORIES`, `name` matches `<category>.<action>`, JSON-serializable `fields`, sensitive-key redaction applied, `serialize_event` deterministic.
- **Non-mutation of inputs** — every supplied input is unchanged after building (deep-equality on copies).
- **Frozen / slotted / closed-shape** — `DailyReport` has `__slots__`, raises `FrozenInstanceError` on assignment, no `__dict__`, exact closed field tuple/types.
- **Equality / identity** — structurally-equal reports/events `==`; distinct `!=`.
- **Composed-pipeline integration** — build a `SignalIntent` (P6-01) → `EligibilityDecision` (P6-02) → `PositionDecision` (P6-03) → `SimulatedOrderIntent` (P6-04) → `SimulatedPortfolio` via `apply_simulated_order_intent` (P6-05), then build and render the daily report, asserting would-trade and at least one would-skip path are summarized and the reconciliation `UNAVAILABLE`/`FAILED` distinction is preserved.
- **Inertness self-check** — AST-walk asserts the closed authorized import set and no forbidden runtime root.
- **Substring self-check** — no `__main__`, `time.sleep(`, `socket.`, `urllib`, `requests.`, `open(`, `uuid.`, `random.`, `logging.basicConfig`, or handler installation.
- **Root-package non-re-export** — `gmc_rebuild` exposes no new P6-06 name; `__all__` excludes it.
- **Package-skeleton-test extension** — `tests/test_package_skeleton.py` extended by exactly one entry (`reporting`).
- **Baseline preservation** — the existing 618 tests continue to pass, plus the new P6-06 tests (count documented).
