### 3.4 Deterministic Identifier Generation Contract (Direction (a) — Planning Level Only)

The candidate deterministic intent-identifier helper has the following contract at the planning level. All elements are planning-level candidates; the implementation authorization names the exact contract.

- **Determinism.** Identical inputs return the byte-for-byte identical identifier `str` every call. No global state, no module-level mutable, no cache, no monotonic counter, no `random`, no `time`, no environment lookup, no clock read, no UUID library usage that consumes entropy or hardware identifiers.
- **Idempotence.** Calling the helper twice with the same inputs returns the same `str`. Calling it `N` times returns the same `str` `N` times.
- **No mutation of inputs.** The supplied closed-input subset is not modified. Repeated calls do not perturb any observable state in any upstream Phase 5 / Phase 6 surface.
- **No hidden state.** The helper holds no module-level mutable state, no class-level mutable state, no instance state. The function is a closed transformation from input values to an output value.
- **No side effects.** No log emission inside the helper itself. No filesystem, no network, no broker SDK, no order placement, no market-data lookup, no time source read, no environment-variable read, no `time.sleep`, no `os.environ` / `os.getenv`.
- **Closed import surface.** The helper's internal module imports only `__future__`, `dataclasses` (stdlib), `enum` (stdlib), `hashlib` (stdlib — only if the implementation authorization chooses a content-hash derivation), and the already-merged in-repo modules required to consume the typed inputs. No import of `gmc_rebuild.config`, `os`, `socket`, `requests`, `urllib`, `http`, `threading`, `asyncio`, `subprocess`, `sqlite3`, `time`, `pickle`, `shelve`, `ssl`, `smtplib`, `ftplib`, `uuid`, `random`, or any third-party dependency.

### 3.6 Safety and Non-Goals

The candidate P6-04 packet explicitly **does not**, regardless of which direction is selected:

- Place any broker call. No broker SDK import, no broker session, no broker reconciliation, no broker credential, no broker reference of any kind. The simulated order intent surface is upstream of any real order; it does not connect to anything.
- Place any real or paper order. No paper-broker connector, no paper-account identifier, no paper-API surface, no paper-trading execution loop. No live-broker connector, no live-account identifier, no live-API surface, no live-trading execution loop.
- Introduce any real venue identifier, real account identifier, real broker identifier, real routing instruction, or real endpoint URL. The merged P5-02 `SimulatedOrderIntent` deliberately omits any venue / broker / account field per its merged authorization; this planning packet preserves that omission, and the implementation authorization is forbidden from re-introducing those fields under direction (a) or direction (b) without a separate explicit written authorization from Kevin per the `RECOVERY.md` §16.5 Risk Register entry on order-intent semantics.
- Mutate any input. The supplied `SimulatedOrderIntent`, `PositionDecision`, `SignalIntent`, `EligibilityDecision`, and `SafetyVerdict` instances are not modified in place. All upstream dataclasses are already frozen / slotted; this packet preserves that.
- Hold any hidden state. No module-level mutable, no class-level mutable, no cache, no counter, no log buffer, no decided-intent queue, no audit ledger.
- Connect to any market. No market-data SDK, no market-data ingestion, no historical data feed, no quote feed, no order book, no FIX / REST / WebSocket client. The `symbol` field of `SimulatedOrderIntent` and `SignalIntent` is a free-form `str` carried forward by value; the helper / composed record does not validate it against any external universe.
- Activate any runtime. No `__main__` entry point, no daemon, no scheduler, no background thread, no long-running service, no `time.sleep`, no `asyncio.sleep`, no cron job, no launchd plist. No re-export from `src/gmc_rebuild/__init__.py`; direction (a)'s public surface remains reachable only via `from gmc_rebuild.simulation import ...`, and direction (b)'s public surface is reachable only via `from gmc_rebuild.<name> import ...`.
- Persist anything. No SQLite, no DuckDB, no on-disk database, no on-disk decided-intent snapshot, no on-disk audit trail, no log file, no file artifact written from any runtime path. No `open(` builtin, no `pickle`, no `shelve`.
