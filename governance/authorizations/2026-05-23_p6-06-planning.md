### 3.1 Candidate Objective

The candidate P6-06 task introduces a **deterministic, in-memory, end-of-cycle daily-report record** that summarizes a single simulated dry-run cycle and is emitted **only** via the merged P2-04 `gmc_rebuild.logging.audit_event` helper to the standard Python logger. Per the merged `docs/FOUNDATION_TO_DRY_RUN_ROADMAP.md` §4 ("Daily report"), `docs/GMC_PROJECT_MAP.md` Layer 2, and `plan/phase6_entry_plan.md` §4 item 6, the report deterministically summarizes, for a given simulated cycle: the intents generated, the decisions made, the simulated fills/applications recorded, the reconciliation status, and any tripped invariants. It is a **pure projection of already-rendered upstream values** (P6-01 `SignalIntent`, P6-02 `EligibilityDecision`, P6-03 `PositionDecision`, P6-04 `SimulatedOrderIntent`, P6-05 `SimulatedPortfolio`, and the merged P4-06 `SafetyVerdict` / P3-05 `ReconciliationStatus` surfaces consumed by value): it computes a structured summary, renders it to an `AuditEvent`, and stops. There is **no external sink, no file artifact, no scheduler, no `__main__`, no persistence, and no runtime activation.** The "report" is a value object plus a deterministic `AuditEvent`; emitting it is the caller's choice of standard-logger handler, exactly as P2-04 already documents.

### 3.7 Deterministic Construction Contract (Planning Level Only)

All elements are planning-level candidates; the implementation authorization names the exact contract.

- **Determinism.** Identical inputs (same intents, decisions, applications, portfolio snapshot, reconciliation status, tripped-invariant set, **and the same explicit caller-supplied timestamp**) return the byte-for-byte identical report record and the byte-for-byte identical `AuditEvent` / serialized JSON every call. No global state, no module-level mutable, no cache, no monotonic counter, no `random`, no UUID-from-entropy.
- **No implicit clock read.** Because `audit_event(...)` falls back to `now_utc()` when `timestamp` is `None`, the candidate P6-06 builder **must pass an explicit caller-supplied timestamp** to keep the record deterministic and avoid a wall-clock dependency. This is the single most important determinism constraint for this capability and the implementation authorization must name it.
- **No mutation of inputs.** The supplied intents, decisions, applications, portfolio snapshot, reconciliation status, and invariant set are not modified in place. All upstream values are already frozen / slotted; this contract preserves that and additionally requires any new report value type to be frozen / slotted.
- **No hidden state and no side effects.** The builder holds no module-level / class-level / instance mutable state, no cache, no counter, no log buffer. It does **not** install logging handlers, call `logging.basicConfig`, write to any sink, touch the filesystem, read env vars, open a socket, or call `time.sleep` / `asyncio.sleep`. Constructing the `AuditEvent` is the function's only output; the *caller* decides whether/where to log it, exactly as P2-04 documents.
- **Closed import surface.** The internal module imports only `__future__`, stdlib `dataclasses` / `enum` / `typing` / `collections.abc` as needed, and the already-merged in-repo modules required to consume the typed inputs and emit the event (`gmc_rebuild.logging`, `gmc_rebuild.signal_intake`, `gmc_rebuild.eligibility`, `gmc_rebuild.decision`, `gmc_rebuild.simulation`, `gmc_rebuild.portfolio_state`, and optionally `gmc_rebuild.runtime` / `gmc_rebuild.risk` / `gmc_rebuild.reconciliation` for the verdict / reconciliation-status types, and `gmc_rebuild.time` only as transitively used by `audit_event`). No import of `gmc_rebuild.config`, `os`, `socket`, `requests`, `urllib`, `http`, `threading`, `asyncio`, `subprocess`, `sqlite3`, `time`, `pickle`, `shelve`, `ssl`, `smtplib`, `ftplib`, `uuid`, `random`, or any third-party dependency.

### 3.9 Proposed Test Matrix and Invariants

The candidate tests are expected, at the planning level only, to cover the following invariant categories. Each is a planning-level candidate; Kevin's future P6-06 implementation authorization may add, remove, or replace any category.

1. **Determinism of the report record and the rendered `AuditEvent`.** Given identical inputs and the same explicit caller-supplied timestamp, the report record, the `AuditEvent`, and its `serialize_event(...)` JSON are byte-for-byte identical across repeated calls. (Mirrors the merged P5-07 / P6-05 determinism precedents.)
2. **Idempotence under repeated rendering.** Rendering the same report twice yields equal `AuditEvent`s; no accumulation, no counter advance. (`plan/phase6_entry_plan.md` §4 item 6 "idempotence under repeated rendering".)
3. **Explicit-timestamp / no-clock-read invariant.** A test asserts the builder requires an explicit timestamp (or, if it permits `None`, that the no-`None` path is the documented deterministic contract) and that two calls with the same explicit timestamp produce identical `AuditEvent.timestamp`. A self-check asserts the module does not call `now_utc()` directly.
4. **Exact match against the `audit_event` structured-record contract.** The rendered event has a category in the closed `AUDIT_CATEGORIES`, a `name` matching `<category>.<action>`, JSON-serializable `fields`, and correct sensitive-key redaction; `serialize_event` is deterministic. (`plan/phase6_entry_plan.md` §4 item 6 "exact match against the `audit_event` structured-record contract".)
5. **Non-mutation of inputs.** Before and after building, every supplied input (intents, decisions, applications, portfolio snapshot, reconciliation status, invariant set) is unchanged (asserted by deep equality on copies).
6. **Frozen / slotted / closed-shape invariants** (only under §3.3 candidate shape (1)). Any new report value type declares `__slots__`, raises `FrozenInstanceError` on post-construction assignment, has no `__dict__`, and exposes exactly the authorized closed field tuple in order with the documented types.
7. **Equality / identity semantics.** Structurally-equal records / events compare `==`; distinct ones compare `!=`; hashability is asserted as inherited or as the explicitly-chosen representation.
8. **Composed-pipeline integration with merged P6-01 / P6-02 / P6-03 / P6-04 / P6-05 surfaces.** End-to-end tests that construct a P6-01 `SignalIntent`, a P6-02 `EligibilityDecision` via `check_eligibility`, a P6-03 `PositionDecision` via `compose_position_decision`, a P5-02 / P6-04 `SimulatedOrderIntent` via `SimulationBoundary.propose_order` (against a clear `SafetyVerdict`), apply it to a P6-05 `SimulatedPortfolio` via `apply_simulated_order_intent`, and then build the daily report over that cycle — asserting the would-trade and at least one would-skip path are summarized correctly, and that the reconciliation status's `UNAVAILABLE`/`FAILED` distinction is preserved in the report. ### 3.10 Safety and Non-Goals (Safety Boundaries)

The candidate P6-06 packet explicitly **does not**, regardless of the chosen location / record / category model:

- Write to any **external sink**. No syslog, journald, cloud logging, HTTP endpoint, message broker, file, socket, or any sink beyond the in-process standard Python logger the caller configures. The report is constructed as an `AuditEvent`; emission is the caller's standard-logger choice, exactly as P2-04 documents.
- Persist anything. No SQLite, DuckDB, Postgres, on-disk database, on-disk report artifact, on-disk audit trail, log file, or any file artifact written from any runtime path. No `open(` builtin, no `pickle`, no `shelve`. (The roadmap §5 and `MASTER_STATUS.md` §6 forbid "generated report artifacts checked into the repository".)
- Activate any runtime. No `__main__` entry point, no `if __name__` block, no daemon, no scheduler, no background thread, no cron / launchd, no long-running service, no `time.sleep`, no `asyncio.sleep`. No re-export from `src/gmc_rebuild/__init__.py`.
- Read the wall clock implicitly. The report timestamp is caller-supplied (§3.7); the builder does not call `now_utc()` / `time.time()` / `datetime.now()`.
- Touch the network. No `socket`, `urllib`, `requests`, `http`, `ssl`, `smtplib`, `ftplib`, or any outbound/inbound network code.
- Maintain any **real position book**, real account, real balance, real holdings, or real P&L. The report summarizes the in-memory P6-05 `SimulatedPortfolio` value object only.
- Place any broker call, real or paper order, market-data lookup, order routing, or venue selection. No broker SDK, no account identifier, no market feed, no fill inferred from any external source.
- Hold any hidden state. No module-level / class-level mutable, no cache, no counter, no buffer.
- Modify any merged surface **except** as explicitly named by the implementation authorization under §3.4 option 2 (a justified, authorized `AUDIT_CATEGORIES` extension with matching P2-04 tests). Absent such an explicit authorization, P6-06 is a pure consumer of the frozen P2-04 `audit_event` contract and the merged P6-01..P6-05 surfaces, modifying none of them.
- Implement strategy logic. The report is a structural projection of already-rendered upstream values to a summary record; it generates no signals, scans no markets, scores no candidates, ranks no alternatives, and picks nothing. Summarization of explicit inputs is bookkeeping, not strategy.
- Load any external configuration. No import of `gmc_rebuild.config`, no `.env` reading, no YAML / TOML / JSON loading from disk. Every input is supplied by the caller.
