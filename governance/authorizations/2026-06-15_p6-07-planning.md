### 3.1 Candidate Objective

The candidate P6-07 task introduces a **deterministic, in-memory, read-only operator view** of the current dry-run engine state. It is a **pure projection of already-rendered upstream values** — the P6-05 `SimulatedPortfolio` snapshot, the cycle's P6-03 `PositionDecision` results, the cycle's P6-04 `SimulatedOrderIntent` values, and the P6-06 `DailyReport` summary — into (a) an immutable operator-facing **value object** and (b) a **deterministic human-readable string render** of that value object. Per `docs/GMC_PROJECT_MAP.md` Layer 2 ("Operator view") and `plan/phase6_entry_plan.md` §46 ("a richer deterministic operator view"), the operator view is the dry-run engine's read-only "dashboard line(s)" — what an operator would read to see, at a glance, the cycle's decisions, the simulated order intents, the end-of-cycle position book, and the daily-report summary including reconciliation status.

It computes a structured view, renders it to a string, and stops. There is **no I/O, no external sink, no file artifact, no scheduler, no `__main__`, no persistence, no network, no live data, and no runtime activation.** The "view" is a value object plus a deterministic `render()` string; what the caller does with that string (print it, log it, show it in a future UI) is the caller's choice and outside this capability. The operator view is **strictly read-only**: it constructs nothing that feeds back into the engine, mutates none of its inputs, and changes no decision, intent, portfolio, or report.

### 3.4 Relationship to the Merged P4-07 `OperatorSafetyView` (Non-Obvious Consideration)

This is the **P6-07 analogue of the P6-06 planning packet's §3.4 audit-category consideration**: a non-obvious decision the implementation authorization must resolve, flagged at planning time so a merged surface is not silently duplicated or confused.

The repository **already has** a read-only operator-facing view — the merged **P4-07 `OperatorSafetyView` / `format_safety_verdict`** at `src/gmc_rebuild/runtime/_operator_view.py` — but it views the inert **`SafetyVerdict`** (heartbeat / kill-switch / reconciliation `CLEAR` vs `BLOCKED`), not the dry-run **engine state** (decisions / intents / portfolio / daily report). The candidate P6-07 operator view is a **distinct, complementary** capability. The implementation authorization must resolve:

1. **Naming.** The P6-07 value object must **not** reuse the name `OperatorSafetyView`. Choose a distinct name (§3.2) so the two views are not confused. (Lower-risk default: a clearly dry-run-scoped name like `DryRunOperatorView`.)
2. **Whether to include the safety verdict.** Kevin's directive lists the **closed inputs** as the P6-03 decisions, P6-04 order intents, P6-05 portfolio snapshot, and P6-06 daily-report summary only — it does **not** list the P4-06 `SafetyVerdict` or the P4-07 `OperatorSafetyView` as inputs. The planning default is therefore to **exclude** the safety verdict from the P6-07 operator view's inputs. ### 3.7 Deterministic Construction Contract (Planning Level Only)

All elements are planning-level candidates; the implementation authorization names the exact contract.

- **Determinism.** Identical inputs (same portfolio snapshot, same decisions, same order intents, same daily report) return the byte-for-byte identical value object and the byte-for-byte identical render string every call. No global state, no module-level mutable, no cache, no monotonic counter, no `random`, no UUID-from-entropy, no set-ordering nondeterminism (any internal ordering is canonicalized deterministically, mirroring `DailyReport`'s sorted/unique discipline).
- **No clock read.** The operator view is a projection of already-rendered values; it takes **no** timestamp of its own and does **not** call `now_utc()` / `time.*` / `datetime.now()`. Any date/time shown is echoed from the caller-supplied `DailyReport.report_date` label (a string), not read from a clock. (Unlike P6-06's `render_daily_report_event`, the P6-07 view does not produce an `AuditEvent` and therefore has no `audit_event` timestamp fallback to guard — it touches no `audit_event` at all; see §3.10.)
- **No mutation of inputs.** The supplied portfolio snapshot, decisions, order intents, and daily report are not modified in place. All upstream values are already frozen / slotted; this contract preserves that and additionally requires the new operator-view value type to be frozen / slotted.
- **No hidden state and no side effects.** The builder/renderer holds no module-level / class-level / instance mutable state, no cache, no counter, no buffer. It does **not** install logging handlers, call `logging.basicConfig`, emit an `audit_event`, write to any sink, touch the filesystem, read env vars, open a socket, or call `time.sleep` / `asyncio.sleep`. Constructing the value object and rendering the string are the function's only outputs.
- **Closed import surface.** The internal module imports only `__future__`, stdlib `dataclasses` / `enum` / `typing` / `collections.abc` as needed, and the already-merged in-repo modules required to type the inputs (`gmc_rebuild.decision`, `gmc_rebuild.simulation`, `gmc_rebuild.portfolio_state`, `gmc_rebuild.reporting`, and `gmc_rebuild.risk` only for the `ReconciliationStatus` type as transitively carried by `DailyReport`). No import of `gmc_rebuild.logging` (the view emits no `audit_event`), no `gmc_rebuild.config`, no `gmc_rebuild.runtime` (no P4-07 composition absent §3.4 authorization), and none of `os`, `socket`, `requests`, `urllib`, `http`, `threading`, `asyncio`, `subprocess`, `sqlite3`, `time`, `pickle`, `shelve`, `ssl`, `smtplib`, `ftplib`, `uuid`, `random`, or any third-party dependency.
- **No re-export from `src/gmc_rebuild/__init__.py`.** The new surface is reachable only via `from gmc_rebuild.operator_view import ...`, mirroring the merged P6-01..P6-06 root-package non-re-export precedent.

### 3.10 Safety and Non-Goals (Safety Boundaries)

The candidate P6-07 packet explicitly **does not**, regardless of the chosen value-object / render model:

- **Activate any runtime.** No `__main__` entry point, no `if __name__` block, no daemon, no scheduler, no background thread, no cron / launchd, no long-running service, no `time.sleep`, no `asyncio.sleep`. No re-export from `src/gmc_rebuild/__init__.py`.
- Read the wall clock. No `now_utc()` / `time.time()` / `datetime.now()`; any date shown is echoed from the caller-supplied `DailyReport.report_date` string.
- Touch the **network**. No `socket`, `urllib`, `requests`, `http`, `ssl`, `smtplib`, `ftplib`, or any outbound/inbound network code.
- Use any **live data**. No real-time feed, no market data, no broker session, no account sync; the view summarizes the in-memory P6-03/P6-04/P6-05/P6-06 value objects only.
- Maintain or display any **real account data**, real position book, real account identifier, real balance, real holdings, or real P&L. The view summarizes the in-memory P6-05 `SimulatedPortfolio` and the P6-06 `DailyReport` value objects only.
- Place any **broker call**, real or paper order, market-data lookup, order routing, or venue selection. No broker SDK, no account identifier, no market feed.
- Hold any hidden state. No module-level / class-level mutable, no cache, no counter, no buffer.
- Modify any merged surface. P6-07 is a pure *consumer* of the merged P6-03 / P6-04 / P6-05 / P6-06 (and transitively `ReconciliationStatus`) surfaces, modifying none of them; it does not compose or modify the merged P4-07 `OperatorSafetyView` absent an explicit §3.4 authorization.
- Implement strategy logic. The view is a structural projection of already-rendered upstream values to a summary value object and string; it generates no signals, scans no markets, scores no candidates, ranks no alternatives, and picks nothing. Read-only summarization of explicit inputs is bookkeeping, not strategy.
- Load any external configuration. No import of `gmc_rebuild.config`, no `.env` reading, no YAML / TOML / JSON loading from disk. Every input is supplied by the caller.
