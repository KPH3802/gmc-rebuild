# P2-03 Authorization — UTC Time Utility

Date: 2026-05-12
Authorizer: Kevin
Scope: Phase 2 / P2-03

## Authorization

I authorize opening **P2-03: UTC Time Utility** as the next Phase 2 work slice for the Grist Mill Capital rebuild.

This authorization is limited to the smallest safe implementation needed to introduce a UTC-only time utility consistent with the project governance model and ADR-004. No trading logic, brokerage integration, market-data integration, scheduling engine, persistence layer, or strategy behavior is authorized by this approval.

## Allowed Scope

P2-03 may include:

- A minimal UTC time utility module.
- Unit tests covering timezone-awareness, UTC normalization, deterministic behavior, and rejection or safe handling of ambiguous/non-UTC inputs.
- Any necessary package exports directly related to the utility.
- Documentation or planning updates required to keep Phase 2 governance artifacts synchronized.

## Required Governance Updates

The P2-03 authorization PR must include:

- This authorization artifact at `governance/authorizations/2026-05-12_p2-03.md`.
- A matching `§8` step-4a allowlist extension for the specific package path needed by P2-03.
- No implementation beyond the authorized authorization/allowlist changes unless explicitly included in a separate P2-03 implementation PR.

## Explicitly Not Authorized

This authorization does not permit:

- Trading signal logic.
- Broker, account, or order-management logic.
- Market-data ingestion.
- Persistent storage or database work.
- Schedulers, background jobs, or live execution loops.
- Expansion into P2-04, P2-05, or later Phase 2 items.

## Exit Criteria

P2-03 may proceed only after the authorization/allowlist change is merged and all repository checks pass.

The implementation PR must remain narrowly scoped and pass the full governance/check suite before merge.
