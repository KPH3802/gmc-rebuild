## Authorized Contract

### 1. Value Type and Builder (Authorized)

- **Type name:** `DryRunOperatorView` (resolves planning §3.4 item 1 — the lower-risk distinct-name default; **not** `OperatorSafetyView`, which is the merged P4-07 surface). Frozen, slotted dataclass with `__slots__`, `FrozenInstanceError` on assignment, no `__dict__`, and an exact closed ten-field tuple.
- **Builder name:** `build_dry_run_operator_view` — pure keyword-only function with no default-mutable arguments.
- **Render:** a `render()` method on the value object (mirrors the merged P4-07 `OperatorSafetyView.render()` precedent) producing a byte-for-byte stable multi-line human-readable string from the value object alone.
- **Location:** new sibling subpackage `src/gmc_rebuild/operator_view/` with tests under `tests/operator_view/`, mirroring the merged P6-01..P6-06 package precedent: an `__init__.py` re-export plus a single internal module `_view.py`.

### 2. Closed Input Set (Authorized — Excludes SafetyVerdict)

- The cycle's P6-03 `PositionDecision` values (a tuple), consumed unchanged.
- The cycle's P6-04 `SimulatedOrderIntent` values (a tuple), consumed unchanged.
- The end-of-cycle P6-05 `SimulatedPortfolio` snapshot, consumed unchanged.
- The P6-06 `DailyReport` summary, consumed unchanged.

The **`SafetyVerdict` is excluded** from inputs (resolves planning §3.4 item 2 in favor of the lower-risk default). Each `PositionDecision` carries a `verdict: SafetyVerdict` field by value, but this packet summarizes only the decision `outcome` and `reasons`; it does **not** re-render the carried verdict. No `gmc_rebuild.runtime` import is authorized; the merged P4-07 `OperatorSafetyView` is **not** composed with by this packet.

### 3. Closed Output (Authorized)

A frozen, slotted `DryRunOperatorView` value object with the closed ten-field set: `report_date` (echoed from `DailyReport.report_date`), `decisions_total` / `would_trade` / `would_skip` (echoed from the daily report; biconditional `would_trade + would_skip == decisions_total` enforced at construction), `decision_lines` (per-decision summary tuple in input order — `"<intent_id>: WOULD_TRADE"` or `"<intent_id>: WOULD_SKIP [<reason>, ...]"`), `order_intent_lines` (per-order-intent tuple `"<symbol> <side> <quantity>"` in input order), `net_position_lines` (per-position tuple `"<symbol>: <signed-net-quantity>"` in `SimulatedPortfolio.positions` canonical order), `applied_intent_ids` (echoed from `SimulatedPortfolio.applied_intent_ids`), `reconciliation_status` (the enum value string echoed from the daily report, preserving the ADR-003 `UNAVAILABLE` vs `FAILED` distinction end-to-end), and `tripped_invariant_lines` (echoed in input order from `DailyReport.tripped_invariants`).

The `render()` method joins these into deterministic multi-line text. No P&L, valuation, cash ledger, balances, broker reconciliation, live account, external state, file artifact, on-disk render, or `AuditEvent` is produced. Hashability is structurally inherited from the carried tuples of strings and ints.

### 4. Deterministic Construction Contract (Authorized)

- **Determinism.** Identical inputs return the byte-for-byte identical `DryRunOperatorView` and the byte-for-byte identical `render()` string every call. No global / module-level / class-level mutable state, no cache, no counter, no randomness, no `uuid`.
- **No clock read.** `build_dry_run_operator_view` and `DryRunOperatorView.render` make no call to `now_utc()` / `time.*` / `datetime.now()`. Any date shown is echoed from the caller-supplied `DailyReport.report_date` label. The module does **not** import `gmc_rebuild.time` (the home of `now_utc`) or stdlib `time`.
- **No mutation of inputs.** The supplied decisions, order intents, portfolio snapshot, and daily report are not modified. All produced values are frozen / value-typed.
- **No hidden state and no side effects.** The builder/renderer holds no module-level / class-level / instance mutable state. No `logging.basicConfig`, no handler installation, no `audit_event` emission, no filesystem write, no env-var read, no socket, no `time.sleep`.
- **Closed import surface.** The internal module imports only `__future__`, stdlib `dataclasses`, and the already-merged in-repo modules required to type the inputs (`gmc_rebuild.decision`, `gmc_rebuild.simulation`, `gmc_rebuild.portfolio_state`, `gmc_rebuild.reporting`, and `gmc_rebuild.risk` for the `ReconciliationStatus` type). **No** `gmc_rebuild.logging` (no `audit_event` emission), **no** `gmc_rebuild.config`, **no** `gmc_rebuild.runtime` (no P4-07 composition; SafetyVerdict excluded per §2), and **none** of `os`, `socket`, `requests`, `urllib`, `http`, `threading`, `asyncio`, `subprocess`, `sqlite3`, `time`, `pickle`, `shelve`, `ssl`, `smtplib`, `ftplib`, `uuid`, `random`, or any third-party dependency.
- **No re-export from `src/gmc_rebuild/__init__.py`.** The new surface is reachable only via `from gmc_rebuild.operator_view import ...`, mirroring the merged P6-01..P6-06 root-package non-re-export precedent.

### 5. §8 Allowlist Impact (Authorized in This PR)

The implementation PR adds **exactly one** new §8 step 4a `allowed_p2_infra` entry (`src/gmc_rebuild/operator_view`), plus its `pr_tag="PR P6-07"` arm, its `for path` iteration entry, its OK-echo `/P6-07` extension, and its comment block — **all in this same PR** per `MASTER_STATUS.md` §8 step 4b. The candidate name `operator_view` does **not** collide with the §8 forbidden-token set (`strategy strategies signal signals scanner scanners model models portfolio backtest backtests broker brokers execution executions live paper daemon daemons market_data order orders secret secrets`), so the §8 step 4 / step 4c scans stay clean with no reconciliation note (a positive contrast to the merged P6-01 `signal_intake` and P6-05 `portfolio_state`).

## Required Tests

At minimum, the implementation PR carries focused deterministic tests covering each of the §3.9 invariant categories enumerated by the merged planning packet:

- **Frozen / slotted / closed-shape invariants.** `DryRunOperatorView` declares `__slots__`, raises `FrozenInstanceError` on assignment, has no `__dict__`, and exposes the exact closed ten-field tuple with the documented types. Validation rejects empty `report_date`, non-int counts, bool counts, negative counts, the `would_trade + would_skip != decisions_total` invariant violation, non-tuple line containers, empty-string line entries, length-mismatched `decision_lines`, and empty `reconciliation_status`.
- **Determinism of the value object and the render string.** Repeated build / render calls with identical inputs return byte-for-byte identical output.
- **Idempotence under repeated rendering.** `view.render()` is referentially transparent for the same view.
- **Read-only / non-mutation of inputs.** Deep-equality on copies confirms the decisions tuple, order-intents tuple, portfolio snapshot, and daily report are unchanged after building.
- **Equality / identity semantics and hashability.** Structurally-equal views compare `==`; distinct views compare `!=`; `hash(view)` returns an `int`.
- **Render-content correctness.** The render string contains the documented `dry_run: ...`, decision counts, reconciliation status, order-intent lines, net-position lines, applied-intent IDs, per-decision detail, and (when present) tripped-invariant lines, in the documented deterministic layout. Empty sections render as `(none)`; the `tripped_invariants:` section is omitted when empty.
- **Composed-pipeline integration with P6-01..P6-06.** End-to-end tests build a `SignalIntent` → `EligibilityDecision` → `PositionDecision` → `SimulatedOrderIntent` → `SimulatedPortfolio` (via `apply_simulated_order_intent`) → `DailyReport` → `DryRunOperatorView`, asserting the would-trade and at least one would-skip path are summarized correctly and the `ReconciliationStatus` `UNAVAILABLE` vs `FAILED` distinction is preserved end-to-end.
- **Distinct from `OperatorSafetyView`.** `DryRunOperatorView is not OperatorSafetyView`; the type names differ; the operator-view source does **not** import `gmc_rebuild.runtime`.
- **AST import-graph inertness self-check.** AST-walk every Python file under `src/gmc_rebuild/operator_view/` asserts imports are drawn only from the closed authorized prefix set (per §4) and that no forbidden runtime root (`os`, `socket`, `requests`, `urllib`, `http`, `threading`, `asyncio`, `subprocess`, `sqlite3`, `pickle`, `shelve`, `ssl`, `smtplib`, `ftplib`, `time`, `uuid`, `random`) is imported.
- **Substring-scan inertness self-check.** Belt-and-suspenders scan for `__main__` blocks, `time.sleep(`, `asyncio.sleep(`, `socket.`, `urllib`, `requests.`, the `open(` builtin, `uuid.`, and `random.` — confirming no sink/handler is installed, no `audit_event` is emitted (proven by absence of the `gmc_rebuild.logging` import), and no runtime activation exists.
- **Root-package non-re-export.** `gmc_rebuild` exposes no new P6-07 public name as an attribute; `gmc_rebuild.__all__` does not include any new P6-07 public name.
- **§8 step 4a allowlist reconciliation.** A test asserts the `allowed_p2_infra` line in `MASTER_STATUS.md` contains `src/gmc_rebuild/operator_view`.
- **Package-skeleton-test extension.** `tests/test_package_skeleton.py` extended by exactly one entry (`operator_view`) with a matching docstring entry naming PR P6-07 and the authorization artifact.
