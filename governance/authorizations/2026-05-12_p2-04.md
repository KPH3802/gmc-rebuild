# P2-04 Authorization — Structured Logging and Audit Event Conventions

Date: 2026-05-12
Authorizer: Kevin
Scope: Phase 2 / P2-04 (governance-only authorization slice)

## Authorization

I authorize opening **P2-04: Structured logging and audit event conventions** as the next Phase 2 work slice for the Grist Mill Capital rebuild, per `plan/phase2_entry_plan.md` §4.

This authorization is a **governance-only slice**, following the same pattern used for P2-03 (`governance/authorizations/2026-05-12_p2-03.md`): the authorizing PR extends the `MASTER_STATUS.md` §8 step 4a allowlist to include the future `src/gmc_rebuild/logging/` submodule and lands this durable authorization artifact. **No implementation, no module files, no tests, and no directory creation are included in the authorizing PR.** The implementation lands in a separate, narrowly scoped P2-04 implementation PR under the exit criteria below.

This authorization is limited to the smallest safe scope needed to introduce structured logging and audit event conventions consistent with the project governance model. No trading logic, brokerage integration, market-data integration, persistence layer, scheduling engine, or strategy behavior is authorized by this approval.

## Allowed Scope

The P2-04 implementation PR, when later opened under this authorization, may include:

- A minimal `src/gmc_rebuild/logging/` submodule containing:
  - A logging configuration module that defines the structured log record shape and log level discipline.
  - An `audit_event` helper that produces structured records with documented fields and documented redaction rules.
  - Output is to the standard Python `logging` framework only; **no external sink, no network transport, no file-rotation daemon, no background flush thread**.
- Unit tests covering log record shape, audit event field set, redaction rules, and the absence of any forbidden runtime behavior.
- Any necessary package exports directly related to the logging/audit utilities.
- Documentation or planning updates required to keep Phase 2 governance artifacts synchronized.
