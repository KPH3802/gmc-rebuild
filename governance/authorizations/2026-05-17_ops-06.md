# OPS-06 Authorization — Recurring Backup Verification & Monitoring Cadence

Date: 2026-05-17
Authorizer: Kevin Heaney
Scope: Bounded, docs/governance-only operations packet for `gmc-rebuild` authorizing the §17 OPS-06 Backup Verification & Monitoring Plan (introduced by GOV-01 at `governance/authorizations/2026-05-17_gov-01.md`) as a standing operator commitment. Converts §17 from "documented plan with recommended cadence" into the operator's recurring manual procedure for backup verification and monitoring. The packet (a) updates `RECOVERY.md` §13 OPS Roadmap to mark OPS-06 as authorized with the cadence and escalation policy now committed; (b) updates `RECOVERY.md` §16.1 Risk Register entry for OPS-06 to reflect the new status and the carry-forward future-control items (alerting / notification policy, automation policy); (c) adds a new `RECOVERY.md` §17.7 "Status as of OPS-06" subsection recording the authorization, the explicit non-changes from §17.6, the re-baseline trigger, the relationship to OPS-05 / OPS-07 / restore drills, the future automation / notification posture, the backup-failure definition, and the §16.1 Risk Register coupling; (d) adds a new entry to `RECOVERY.md` §12 References pointing at this authorization artifact; and (e) adds this durable authorization artifact at `governance/authorizations/2026-05-17_ops-06.md`. Implements no automation, no scheduler, no daemon, no cron job, no launchd plist, no alerting system, no notification mechanism. Makes no changes to any source, test, monitoring, ADR, `MASTER_STATUS.md`, `plan/phase4_entry_plan.md`, `AI_WORKFLOW.md`, `pyproject.toml`, `.pre-commit-config.yaml`, `.secrets.baseline`, `.gitignore`, or any other `governance/authorizations/*.md` file. Makes no setting change to Backblaze, Time Machine, LaCie, X10 Pro, FileVault, ownership, sleep / power / USB / network / DNS, or any other macOS configuration surface. Does not run any destructive backup test, any restore execution, any drive action, or any system change. Does not promote the X10 Pro to Layer 5 (OPS-05 remains future / not authorized). Does not authorize any Phase 5 execution-adjacent work or any P5-03+ implementation. Defines the cadence and policy as an operator commitment; each individual check run and each individual restore drill remains an operator-side action requiring no per-execution authorization beyond OPS-06 itself for the §17.2 read-only inspection set, but **explicitly** requires its own separate written authorization for any drill recorded under §5.5 / §5.6 (consistent with the OPS-02 / OPS-04B precedent), for any incident-triggered investigation packet (OPS-03 shape), and for any future automation or notification mechanism.
