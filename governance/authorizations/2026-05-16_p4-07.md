## Allowed Scope

The implementation PR may include exactly:

- A new source module `src/gmc_rebuild/runtime/_operator_view.py` containing only a read-only operator-facing formatter (`OperatorSafetyView` frozen dataclass and the pure `format_safety_verdict` function) that converts an existing `SafetyVerdict` (produced by the merged P4-06 `RuntimeShell`) into deterministic human-readable lines. The module has no `__main__`, no daemon, no scheduler, no network, no persistence, no broker, no market data, no order placement, no strategy/scanner/model/portfolio/backtest/live/paper trading, no env-var or secrets loading, and no `time.sleep`.
- A narrow update to `src/gmc_rebuild/runtime/__init__.py` re-exporting the new symbols.
- A new test module `tests/runtime/test_operator_view.py` with focused tests proving the view is read-only, deterministic, local, and safety-preserving (cannot widen a blocked verdict to clear).
- This authorization artifact at `governance/authorizations/2026-05-16_p4-07.md`.
- Conservative status reflections in `MASTER_STATUS.md` and `plan/phase4_entry_plan.md` recording P4-07 as a narrow read-only operator-view task that lives entirely within the existing `src/gmc_rebuild/runtime/` directory and does not extend the §8 step 4a allowlist.

The new operator-view module may:

- Accept an already-built `SafetyVerdict` by value and return an immutable `OperatorSafetyView`.
- Reuse the merged P4-06 `SafetyVerdict` type, the merged P2-05 risk-control enums (`HeartbeatStatus`, `KillSwitchState`, `ReconciliationStatus`) — accessed through the `SafetyVerdict` fields without further imports — and standard-library inert primitives (`__future__`, `collections.abc`, `dataclasses`, `types`).
