## Closed Inputs / Outputs

- **New flag:** `--emit-reconciliation-json PATH|-`. Supported **only** with
  `--source insider_cluster` (rejected with a non-zero argparse error on the
  synthetic source, mirroring `--emit-json`). When the argument is a filesystem
  path, the reconciliation JSON is written to that single caller-supplied path
  (the parent directory must already exist; no directory creation). When the
  argument is the single character `-`, the JSON is written to stdout after the
  human summary; no file is created. No flag = no reconciliation emission and no
  file write. The flag is **independent** of `--emit-json`: either, both, or
  neither may be passed.
- **Reconciliation performed:** a pure, deterministic, read-only
  **self-comparison** — `reconcile_dry_run_positions(simulated=portfolio,
  expected=ExpectedPositions.from_simulated_portfolio(portfolio),
  reconciliation_status=report.reconciliation_status)`. By construction this
  yields `outcome=MATCH` with every simulated position in `matches` and empty
  difference lists. The `reconciliation_status` is echoed verbatim from the
  cycle's `DailyReport`. No real account, no broker, no external expected-
  positions source.
- **Output shape:** exactly the P6-10
  `build_dry_run_reconciliation_json_payload` dict, pretty-printed with
  `indent=2, sort_keys=True` for stable text diffs (same rendering convention
  as `--emit-json`).
