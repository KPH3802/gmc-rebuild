## Allowed Scope

The P4-03 implementation PR, under this authorization, may include exactly:

- A new test module at the exact path `tests/p4_02_composed/test_composed_invariants.py` containing focused, deterministic tests for **invariants over composed scenarios** asserting:
  - properties that hold across multiple composed scenarios driven by an in-memory fixed clock — for example, that `KillSwitchState.TRIPPED` is sticky across subsequent `current()` calls regardless of heartbeat freshness or reconciliation outcome; that `HeartbeatStatus.STALE` is independent of `KillSwitchState` and `ReconciliationStatus`; that `ReconciliationStatus.UNAVAILABLE` is preserved across kill-switch trips and heartbeat staleness; that ADR-004 UTC-Z-suffixed timestamps hold at every composed-fixture boundary regardless of which fake's clock has advanced;
  - structural invariants exercised across the three already-merged abstract `Protocol`s (`HeartbeatProtocol`, `KillSwitchProtocol`, `ReconciliationProtocol`) declared in `src/gmc_rebuild/risk/`;
  - import-shape invariants matching the merged P4-02 composed-fixture pattern (no forbidden runtime imports leak into the test module; imports come only from the three merged Phase 3 fixture packages, `gmc_rebuild.risk`, and the standard-library helpers `ast`, `datetime`, `pathlib`, `__future__`);
  - no broker / network / filesystem / scheduler / market-data / order / position / fill / execution / persistence coupling anywhere in the test or its imports.
- The test module avoids `pytest.raises` and fixture-typed parameters to remain compatible with the mypy-strict pre-commit hook (the same pattern P3-03 / P3-04 / P3-05 / P4-02 tests already follow per `tests/test_risk_interfaces.py` and `tests/p4_02_composed/test_composed_fixture.py`).
