### 3. Closed Value-Object Shapes (Authorized — Exact Fields, in Order)

All three dataclasses are declared `@dataclass(frozen=True, slots=True)`, raise `FrozenInstanceError` on post-construction assignment, and have no `__dict__`.

**`DryRunReconciliationOutcome(StrEnum)`** — exactly two members, no third:
- `MATCH = "MATCH"`
- `MISMATCH = "MISMATCH"`

**`ReconciliationQuantityMismatch`** — exactly three fields, in order:
- `symbol: str` — non-empty.
- `simulated_quantity: int` — signed `int`, rejects `bool`.
- `expected_quantity: int` — signed `int`, rejects `bool`.
- `__post_init__` enforces `simulated_quantity != expected_quantity` (an equal pair belongs under `matches`, not `quantity_mismatches`).

**`ExpectedPositions`** — exactly one field:
- `positions: tuple[tuple[str, int], ...]` — canonical: each pair is `(non-empty symbol, signed non-zero int quantity)`; sorted by symbol ascending; symbols unique; no zero-quantity entry. `__post_init__` enforces all of these and rejects `bool` quantities.
- One authorized classmethod constructor: `from_simulated_portfolio(cls, snapshot: SimulatedPortfolio) -> ExpectedPositions` — synthesizes an `ExpectedPositions` from a merged P6-05 `SimulatedPortfolio` (for tests and self-comparison sanity checks). It is implemented in terms of the `SimulatedPortfolio` value object only; it opens **no** real-account / broker / live-feed path. No `from_broker(...)` / `from_account_snapshot(...)` constructor is authorized.

**`DryRunReconciliationResult`** — exactly six fields, in order:
- `outcome: DryRunReconciliationOutcome`.
- `matches: tuple[tuple[str, int], ...]` — canonical (sorted by symbol, unique, non-zero) `(symbol, net_quantity)` pairs present in **both** snapshots with equal signed net quantities.
- `quantity_mismatches: tuple[ReconciliationQuantityMismatch, ...]` — canonical (sorted by symbol, unique) records where both snapshots carry an entry but the quantities differ.
- `only_in_simulated: tuple[tuple[str, int], ...]` — canonical `(symbol, net_quantity)` pairs present in the `SimulatedPortfolio` but absent from `ExpectedPositions`.
- `only_in_expected: tuple[tuple[str, int], ...]` — canonical `(symbol, net_quantity)` pairs present in `ExpectedPositions` but absent from the `SimulatedPortfolio`.
- `reconciliation_status: ReconciliationStatus` — caller-supplied; echoed verbatim from the merged `gmc_rebuild.risk.ReconciliationStatus` enum, preserving the ADR-003 `UNAVAILABLE` vs `FAILED` distinction end-to-end.
- `__post_init__` enforces the biconditional `outcome == MATCH iff (quantity_mismatches == () and only_in_simulated == () and only_in_expected == ())`, and enforces the canonical-form invariants on every tuple field.

## Required Tests / Invariants

The implementation PR carries a focused, deterministic tripwire suite under `tests/dry_run_reconciliation/test_dry_run_reconciliation.py` covering at minimum the following invariant categories (mirroring the planning packet §3.9 matrix; the exact new-test count is documented in the implementation PR and the existing 766-test baseline is preserved):

1. **Determinism / idempotence.** Identical inputs return a byte-for-byte identical result across repeated calls; no accumulation, no counter advance, no perturbation of inputs.
2. **Read-only / non-mutation of inputs.** `SimulatedPortfolio`, `ExpectedPositions`, and `ReconciliationStatus` are unchanged before and after reconciling.
3. **Frozen / slotted / closed-shape invariants.** `DryRunReconciliationResult`, `ExpectedPositions`, and `ReconciliationQuantityMismatch` each declare `__slots__`, raise `FrozenInstanceError` on assignment, have no `__dict__`, and expose exactly the authorized closed field tuple in order.

5. **Canonical-form invariants** on `matches` / `only_in_simulated` / `only_in_expected` (sorted by symbol, unique, no zero-quantity entry) and on `quantity_mismatches` (sorted by symbol, unique, each `simulated_quantity != expected_quantity`).
6. **Comparison correctness.** Symbol present in both with equal quantity → `matches`; present in both with differing quantity → `quantity_mismatches`; present only in simulated → `only_in_simulated`; present only in expected → `only_in_expected`; the all-empty case → `MATCH`.
