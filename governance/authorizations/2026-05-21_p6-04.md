## Authorized Contract

The P6-04 Direction A surface is authorized with the following contract:

- **Closed `SimulatedOrderTimeInForce` enumeration.** A `StrEnum` with exactly four members — `DAY`, `GOOD_TILL_CANCEL`, `IMMEDIATE_OR_CANCEL`, `FILL_OR_KILL` — drawn from the planning packet's candidate member set. The enum is a pure data tag; it triggers no broker action and carries no execution semantics. Any future member requires its own separate written authorization per `AI_WORKFLOW.md` §7.
- **Ninth `SimulatedOrderIntent` field.** `time_in_force: SimulatedOrderTimeInForce`, defaulting to `SimulatedOrderTimeInForce.DAY`. The field is validated at construction (`__post_init__`) to be a `SimulatedOrderTimeInForce` member. The dataclass remains frozen and slotted; immutability is preserved. The field carries no venue, account, broker, routing, persistence, or execution capability.
- **Deterministic identity helper.** A pure function deriving a deterministic identifier from a closed subset of the order-intent's content (lane, symbol, side, quantity, order type, limit price, time-in-force, created-at timestamp). Identical inputs return the byte-for-byte identical identifier; no randomness, no clock read, no counter, no hidden state, no side effects. The helper reads its `created_at` from the caller-supplied value; it does not read a wall clock.
- **Preserved P5 / P6 behavior.** The merged `SimulationBoundary.propose` (P5-01) and `SimulationBoundary.propose_order` (P5-02) identity-return contracts, the `SafetyVerdict.clear` precondition, the lane-mismatch and blocked-verdict rejection behavior, and the merged P6-01 / P6-02 / P6-03 surfaces are preserved byte-for-byte semantically. Existing eight-argument `SimulatedOrderIntent` construction continues to work because `time_in_force` is defaulted.
- **Inert by construction.** No `__main__`, no daemon, no scheduler, no background thread, no long-running service, no broker integration (real or paper), no market-data ingestion, no order placement, no order routing, no execution adapter, no strategy / scanner / model / portfolio / backtest content, no env-var loading, no secrets, no network, no persistence, no `time.sleep`, no `asyncio.sleep`. The identity helper's only new import is `hashlib` (stdlib), explicitly authorized for the content-hash derivation by the planning packet §3.4.
