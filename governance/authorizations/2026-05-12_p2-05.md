# P2-05 Authorization — Risk-Control Interfaces (No Broker)

Date: 2026-05-12
Authorizer: Kevin
Scope: Phase 2 / P2-05 (governance-only authorization slice)

## Authorization

I authorize opening **P2-05: Risk-control interfaces (no broker)** as the next Phase 2 work slice for the Grist Mill Capital rebuild, per `plan/phase2_entry_plan.md` §4.

This authorization is a **governance-only slice**, following the same pattern used for P2-03 (`governance/authorizations/2026-05-12_p2-03.md`) and P2-04 (`governance/authorizations/2026-05-12_p2-04.md`): the authorizing PR extends the `MASTER_STATUS.md` §8 step 4a allowlist to include the future `src/gmc_rebuild/risk/` submodule and lands this durable authorization artifact. **No implementation, no module files, no tests, and no directory creation are included in the authorizing PR.** The implementation lands in a separate, narrowly scoped P2-05 implementation PR under the exit criteria below.

This authorization is limited to the smallest safe scope needed to introduce abstract risk-control interfaces consistent with the project governance model and ADR-002 (kill switch), ADR-003 (reconciliation), and ADR-005 (heartbeat). No trading logic, brokerage integration, market-data integration, persistence layer, scheduling engine, order-management surface, or strategy behavior is authorized by this approval.

## Allowed Scope

The P2-05 implementation PR, when later opened under this authorization, may include:

- A minimal `src/gmc_rebuild/risk/` submodule containing:
  - Abstract `Protocol` / interface definitions (and any supporting frozen dataclasses required by the interface signatures) for the kill switch, reconciliation, and heartbeat boundaries described in ADR-002, ADR-003, and ADR-005.
  - **Types and abstract boundaries only.** No concrete implementation that talks to a broker, account, market-data source, network, filesystem, or any external system. No `__main__` entry point, no background thread, no scheduler, no daemon. No instantiation of a real or simulated execution loop.
- Unit tests covering protocol shape, dataclass immutability, the absence of any forbidden runtime behavior, and (optionally) **test-only** fake / stub implementations of the protocols that exist solely to exercise the interfaces in tests. Test-only fakes must live under `tests/`, must not be exported from the runtime package, must not place any order, must not touch a network or filesystem, and must not pretend to be a real broker.
- Any necessary package exports directly related to the risk-control interfaces (types and protocols only).
- Documentation or planning updates required to keep Phase 2 governance artifacts synchronized.

The P2-05 authorization PR must include:

The §8 step 4a allowlist after this PR therefore contains exactly five entries: `src` (PR P2-01), `src/gmc_rebuild/config` (PR P2-02), `src/gmc_rebuild/time` (PR P2-03 authorization slice), `src/gmc_rebuild/logging` (PR P2-04 authorization slice), and `src/gmc_rebuild/risk` (PR P2-05 authorization slice). All five are governance-bounded infrastructure paths and none is a forbidden category under §8 step 4 / step 4c.

## Explicitly Not Authorized

The authorization above does not permit, by implication or otherwise, any of the following. Each remains forbidden until Kevin records a separate written authorization per `MASTER_STATUS.md` §7 and `AI_WORKFLOW.md` §6 rule 3:

- Trading signal logic, scanners, models, portfolio rules, or backtests.
- Broker, account, or order-management logic, including broker SDK integration, order objects, order placement, position management, fills, or trade reports.
- Live trading workflows, or paper-trading workflows wired to a real broker.
- Market data ingestion, real data pipelines, or stored datasets.
- Persistent storage or database work of any kind (no SQLite, no on-disk reconciliation snapshots, no on-disk heartbeat state by P2-05 code).
- Schedulers, background jobs, long-running services, live execution loops, or operator-heartbeat daemons. No `if __name__ == "__main__"` entry point under `src/gmc_rebuild/risk/`. The kill-switch / heartbeat / reconciliation interfaces are defined as types and protocols only; their runtime wiring is out of scope.
- Loading configuration or secrets from environment variables (`os.environ`, `os.getenv`, `.env` files) inside the risk submodule.
- Real secrets, private keys, certificates, real account identifiers, real broker / venue names, or real endpoints embedded in default protocol fields, dataclass fields, or interface identifiers.
- Wiring any risk-control interface to a real or simulated broker, exchange, message bus, webhook, syslog/journald, cloud logging, or any other external sink.
- Any concrete implementation of the kill-switch, reconciliation, or heartbeat protocols inside the runtime package. Test-only fakes are permitted under `tests/` as described in "Allowed Scope" and are not part of the runtime package surface.
- Any submodule under `src/gmc_rebuild/` beyond the `risk/` directory named above.
- Any relaxation of pre-commit, Ruff, mypy strict mode, detect-secrets, `.gitignore`, `.secrets.baseline`, the `MASTER_STATUS.md` §8 step 4 always-forbidden scan, the §8 step 4c recursive forbidden-token scan, or any other quality gate.
- Any new top-level directory.
- Expansion into any Phase 2 task beyond P2-05 or any Phase 3+ item.
- Any code in the authorizing PR itself: the directory is allowlisted but not materialised, and no `.py` files land under `src/gmc_rebuild/risk/` in this PR.

This list mirrors the always-forbidden categories in `MASTER_STATUS.md` §6 and `plan/phase2_entry_plan.md` §2. The mirroring is intentional and not a re-authorization: those categories remain forbidden by default, and naming them here records that Kevin did not silently relax them when authorizing the P2-05 slice.
